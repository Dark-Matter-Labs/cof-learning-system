Make two changes to layout.tsx:

**Change A — Remove static className="dark" from html, replace with system-preference script:**

Remove `className="dark"` from the `<html>` element. Add a `<script>` tag as the FIRST child of `<head>` (or first child of body before any rendering) that reads `window.matchMedia('(prefers-color-scheme: dark)').matches` and toggles the 'dark' class on `document.documentElement` before the page paints. Use a dangerouslySetInnerHTML inline script to avoid FOUC.

The script content (pass as dangerouslySetInnerHTML):
```javascript
(function(){var d=document.documentElement;if(window.matchMedia('(prefers-color-scheme: dark)').matches){d.classList.add('dark')}else{d.classList.remove('dark')}})();
```

Place the script inside `<head>` using Next.js `<Script strategy="beforeInteractive">` OR as a raw `<script dangerouslySetInnerHTML>` in the `<html>` children before `<body>`.

**Change B — Add padding-top to main:**

Change `<main className="h-screen overflow-y-auto">` to `<main className="h-screen overflow-y-auto pt-[49px]">`.

This ensures non-graph pages (capture, review, reflect, settings) have content starting below the 49px fixed navbar. The graph page overrides this with its own full-screen absolute layout.

Note: The graph page's root div is `w-full h-screen relative overflow-hidden` — it fills the entire padded main and overrides with absolute positioning. This is fine — the graph canvas has its own top-[49px] offsets for panels.

Final html element: `<html lang="en">` (no className — class is set by the script).
